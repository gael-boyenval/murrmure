# Manifest and bundle schema (v1)

**Status:** normative (2026-06-21)  
**Supersedes:** P5 `mount_export`-only manifest for new capabilities  
**CDK package:** `@studio/capability-sdk` `CapabilityManifestSchema`

---

## Manifest schemaVersion `"1"`

```typescript
export const CapabilityManifestSchema = z.object({
  schemaVersion: z.literal("1"),
  id: z.string().regex(/^[a-z][a-z0-9-]{1,62}$/),     // package_id
  version: z.string(),                                  // semver
  routes_prefix: z.string().regex(/^\/api\/[a-z0-9-]+$/),
  ui: z.object({
    entry: z.string(),                                  // relative → bundled ui/entry.js
    canvas_route: z.string(),                           // e.g. /spaces/:spaceId/sessions/:sessionKey
    shell_html: z.string().optional(),                  // default: ui/shell.html (iframe host page)
  }),
  server: z.object({
    mount_module: z.string(),                             // relative → server/mount.mjs
  }),
  mcp_tools_by_version: z.record(z.string(), z.array(z.string())), // semver key → tool names
  config_schema: z.string().optional(),                 // contract/config.schema.json
  tests: z.object({ contract: z.string().optional() }).optional(),
  // contract_ref_id: NOT in author manifest — hub assigns at ingest (ARCH-03)
});
```

### Migration from P5

| P5 field | v1 field |
|----------|----------|
| `mount_export` | `server.mount_module` (path in bundle, not npm package name) |
| `mcp_tools: string[]` | `mcp_tools_by_version[version]` |
| `contract_ref_id` | **removed from author manifest** — hub assigns |

## Bundle tree (required files)

After `studio capability build`, staged tree **must** contain:

```
{stage}/
  manifest.json              # resolved paths (bundled-relative)
  contract/
    contract.json
    config.schema.json       # if manifest.config_schema set
    mcp-tools.json           # tool schemas + route map
  ui/
    shell.html               # iframe bootstrap; loads entry.js
    entry.js                 # ESM; exports mount()
    assets/…
  server/
    mount.mjs                # ESM; exports mountRoutes()
  tests/                     # optional
  bundle.digest              # sha256:… (see below)
  build.meta.json
  .push-state.json           # written on push (CLI), not in tarball
```

---

## Deterministic digest

| Rule | Value |
|------|-------|
| Algorithm | SHA-256 over canonical tar.zst bytes |
| Canonicalization | Sorted file paths; normalized mtimes stripped; UTF-8 JSON keys sorted |
| Prefix | `sha256:` + lowercase hex |
| Excluded from hash | `.push-state.json`, `build.meta.json`, `bundle.digest`, `bundle.tar.zst` (metadata sidecars) |

Hub **recomputes** digest on ingest; rejects if client `bundle_digest` mismatch (ARCH-04).

## build.meta.json

```json
{
  "package_id": "review-loop-lite",
  "version": "1.0.0",
  "bundle_digest": "sha256:…",
  "source_path": "/Users/dev/my-workflows/review-loop-lite",
  "built_at": "2026-06-20T12:00:00Z",
  "sdk_version": "0.1.0",
  "ui_framework": "vite-react"
}
```

Configure install detail displays `source_path`, `bundle_digest`, `built_at` (UX-05).

In v1, `ui_framework` is constrained to the React build profile (`"vite-react"`) for scaffolds generated by `studio capability init`.

## contract/mcp-tools.json

```json
{
  "tools": {
    "create_review_session": {
      "http": { "method": "POST", "path": "/sessions" },
      "input_schema": { "type": "object", "properties": { … } }
    }
  }
}
```

Lens A: every name in `mcp_tools_by_version[manifest.version]` must exist in `tools`.

Hub MCP rebuild uses these schemas — see [07-mcp-tool-model-and-catalog-rebuild.md](./07-mcp-tool-model-and-catalog-rebuild.md).

## Minor vs major semver (MCP tools)

| Bump | `mcp_tools_by_version` |
|------|------------------------|
| Patch | Same tool list |
| Minor | Add tools; optional new keys in schemas |
| Major | May remove/rename tools; new `contract_ref_id` major from hub |

Runtime honors `mcp_tools_by_version[live semver]` on catalog rebuild.

## Validation (offline)

CDK `validateCapabilityRoot` blocking:

- Manifest schema
- Bundle tree completeness (post-build)
- Contract graph reachability
- Tool ↔ route map consistency
- `routes_prefix` / `canvas_route` syntax
